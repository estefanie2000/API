# API
Este proyecto consiste en la automatización de pruebas de API utilizando Python.
El objetivo es validar el funcionamiento de diferentes endpoints de una API pública mediante casos de prueba automatizados, simulando escenarios reales de testing.

El proyecto fue desarrollado como parte de mi proceso de formación en QA, con el propósito de fortalecer mis habilidades en automatización, pruebas de API y buenas prácticas de testing.

Objetivo

Automatizar pruebas sobre endpoints de una API.
Validar códigos de estado HTTP.
Verificar la estructura y contenido de las respuestas JSON.
Implementar escenarios positivos y negativos.
Aplicar una estructura básica de framework de automatización.



El proyecto incluye:
Creación de usuarios mediante requests POST.
Consulta de usuarios mediante requests GET.
Validación de códigos de estado (200, 201, 404).
Verificación de campos clave en las respuestas (name, email, etc.).
Ejecución de múltiples casos de prueba.
Manejo de errores y respuestas no JSON.
Separación de la lógica del cliente API y los tests.
